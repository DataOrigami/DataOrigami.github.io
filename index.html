Mixology: Data-driven drinks for our first meeting

It's our first meeting, so I think we should start with something unique. Every proper club needs a proper drink.

Can we datamine a unique club drink using a large dataset of cocktail drinks?

Yes, probably.

When: Tuesday, Sept. 17th, 6:30 - 9pm

Where: Shopify Lounge. 126 York St. You will need to go through the parking lot to get in by the back. We have some glass doors beside the loading dock for Market Organics. We'll be letting people in from there.

Who: Data hackers, designers, programmers, stats/math/CS majors -- if you are looking to hack (though you don't need to), you should know a programming language. Hosted by Cam DP.

Goals

Determine the ingredients: (and perhaps the directions to make the cocktail, too) This may require an objective function. Using the database to weed out bad combinations is a win too! Ideally the drink should have some flair to it.

Create a name for the cocktail: (this should be data determined too) Naively, gather all names of cocktails, split on whitespace, and randomly pick. Better: considering the ingredients we have chosen, look for a pair or triplet of existing cocktails that can be "summed up" to produce our cocktail. This is like a regression task. Using Google Trends to find "hot names", i.e. "One VMA Miley" would be something hilarious.

Look for interesting relationships in the dataset: orange juice => vodka? Some algos that might help: association rules, network analysis...

Create a cool postcard-sized graphic about our cocktail: (with recipe) We'll use this to create actual postcards to promote our club: "Dear P.M. Harper, we've noticed you could use a drink...". "Dear Ottawa Citizen, we have provided for you a result you might finding interesting-- the url is at ...."

Can we light it on fire? Why not?

Actually drink it. Can we get a local drinking establishment to serve it? Maybe exclusively to our members/meetings?

Our code/dataset can be found on Github

6:30-9pm, Sept. 17th, 2013
